2. Sees role selection page with 3 buttons
3. Clicks their role (Student/Teacher/Staff)
4. Form for that role appears (no page reload)
5. Fills in credentials and submits
6. Django processes form (POST request)
7. Authenticates user
8. Redirects to dashboard or shows error

REGISTRATION FLOW:
1. User visits /register/{role}/
2. Sees registration form with 10-15 fields
3. Fills in all required fields
4. Submits form
5. Django validates data
6. Creates user account and profile
7. Shows success message
8. Redirects to login page

   by loading options from the database in your views.

A: Send an email with verification link after registration.
   Mark email as verified only after user clicks link.

Q: How do I add password reset?
A: Create a forgot_password view. Send email with reset token.
   Create reset_password view to handle token verification.

Q: Can I customize the form fields?

Q: How do I add client-side validation?
